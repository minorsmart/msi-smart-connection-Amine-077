In week 3 wordt de webpagina verder uitgebreid door het gebruik van JavaScript, dit geeft je de mogelijkheid om interactie op je pagina toe te voegen wat de mogelijkheden van verschillende vormen van weergave enorm verbreedt. Bijvoorbeeld de collapsibles die deze pagina overzichtelijker maken werken aan de hand van een stukje JavaScript, de code die hiervoor gebruikt is is als volgt:

var coll = document.getElementsByClassName("collapsible"); var i;

{
coll[i].addEventListener("click", function() {
this.classList.toggle("active");
var content = this.nextElementSibling;

} else {

In de eerste regel wordt de opdracht gegeven om alle elementen die het "class" label hebben te verzamelen. Ook wordt er aangegeven dat de variabele in dit script met 'i' zal worden weergegeven.
In de tweede regel wordt weergegeven dat de functie die in regel 3 staat weergegeven geactiveerd moet worden wanneer er op een coll (collapsible) wordt geklikt.
De functie die geactiveerd wordt is dat de layout van de collapsible veranderd naar klasse "active" uit het css bestand 'Opmaak.css'.
Het klikken activeert ook dat de "nextElementSibling" wordt weergegeven, in dit geval is dat de paragraaf tekst die wordt weergegeven wanneer er op een collapsible wordt geklikt.
In de if/else weergave wordt de code zo ingericht dat de uitgeklapte collapsible altijd de hoogte van de tekst die weergegeven wordt aanneemt +"px" is nodig zodat de marge er omheen nog wordt meegenomen.

In week 4 ging de les over databases, de soorten databases en de manier waarop deze databases gebruikt kunnen worden. Databases kunnen opgedeeld worden in 2 grote groepen:

  1. SQL of relational databases
    Deze databases zijn de populairste databases, dit heeft voornamelijk te maken met het feit dat deze makkelijker te gebruiken zijn en een stuk beter te begrijpen zijn. Een relationele database is een database die bestaat uit tabellen, elk met een schema waarin de data gepresenteerd wordt. De tabellen hebben allen een key die wordt gebruikt om specifieke tabellen, kolommen of rijen te identificeren en hier sneller toegang tot te krijgen. SQL of Structured Query Language databases hebben als voordeel dat deze zeer goed gestructureerd zijn. Dit zorgt ervoor dat deze databases eenvoudig te gebruiken zijn, hierdoor zijn er veel opgeleide mensen die met deze databases kunnen werken. Het nadeel van deze databases is dat ze niet werken met slecht gestructureerde data, dit maakt het lastig om dit databasetype te gebruiken voor bijvoorbeeld analyses van nieuwe data.
  2. NoSQL of Non-relational databases
    Dit type database is het populairste alternatief voor de SQL databases. Bij dit type database kan de database uit allerlei vormen bestaan in tegenstelling tot de SQL database. Dit heeft als groot voordeel dat de NoSQL database zeer wijd ingezet kan worden en flexibeler is in de toepassingen en het gebruik. Ook zijn NoSQL databases toleranter met foutjes in de databases. Echter hebben ook NoSQL databases een aantal nadelen, zo zijn deze databases door de vrijheid in gebruik ook lastiger toe te passen. Hierom worden deze databases nog minder de gebruikt dan SQL databases, echter is het gebruik van NoSQL steeds meer op komst wat duidt op een breder toepassingsveld.

In week 5 ligt de focus op het toevoegen en gebruiken van API's. Hiervoor hebben we tijdens de les gewerkt aan het laten werken van een zogenoemde weather API. Dit is een API die het weer met een beschrijving en temperatuur op de gekozen locatie live weergeeft. Hiervoor wordt gebruik gemaakt van openweathermap.org om de API key op te halen. Wanneer de API key is opgehaald kan deze gebruikt worden in onderstaande code.

function weatherBalloon( cityID )
{ var key = '93012064e501c3c9b57a3579a10fbf0f';
fetch('https://api.openweathermap.org/data/2.5/weather?id=' + cityID+
'&appid=' + key)
.then(function(resp) { return resp.json() })
.then(function(data) {
drawWeather(data);
})
.catch(function() {
});
}

window.onload = function() {
weatherBalloon( 2746301 );
}
function drawWeather( d ) {
var celcius = Math.round(parseFloat(d.main.temp)-273.15);
document.getElementById('description').innerHTML = d.weather[
0].description;
document.getElementById('temp').innerHTML = celcius + '°';
document.getElementById('location').innerHTML = d.name;
}

De var key die zichtbaar is in de tweede regel is de key die van openweathermap.org is opgehaald om gebruik te kunnen maken van deze API. Het fetch command wordt gebruikt om de resources van de url in het command op te halen. Deze resources worden door middel van de .then() functie omgezet naar een JSON response. Deze wordt gekoppeld aan een weerballon id. In de drawweather functie wordt de data die vanuit de weerballon wordt opgehaald weergegeven per id.
